## Architecture notes

### Streaming architecture

Sprint 2 introduces end-to-end streaming:
- **Backend:** FastAPI SSE endpoint with Accept-header negotiation
- **Workflow:** Generator-based event emission at executor boundaries
- **Frontend:** TypeScript SSE service with typed event models
- **UX:** Per-executor progress tracking (Intent → SQL → Executing → Result)

### Security architecture

Reinforced by user directives:
- Frontend → APIM → Backend (no direct frontend-backend communication)
- APIM as security gateway (JWT validation, claim extraction, header injection)
- MSAL-based authentication in Angular frontend
- Container Apps infrastructure ready (ACA) for deployment

### Frontend architecture

Design-first approach:
- Match Calabrio Supervisor Assist visual reference (not generic chatbot)
- MSAL authentication integration
- SSE streaming for real-time progress
- Mobile-first responsive design
- Light/dark theme support
- Animated progress indicators
